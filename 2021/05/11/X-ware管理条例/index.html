X-ware管理条例

序言：X-ware的存在和价值

X-ware是SNC传统的活动之一，是为全校师生提供计算机技术支持的平台。它服务的内容，涵盖了计算机的硬件（Hardware）和软件（Software）两个方面，这也是其名称“X-ware”的由来。
X-ware服务的价值，不仅仅在于使广大师生获得便利，更重要的是锻炼系统维护组成员的实践动手能力。通过这样一项服务，我们的成员确实得到了更多的实践机会，也在实践过程中的对系统维护中一些知识和思想有了更加深刻的理解。
总而言之，无论对内还是对外，X-ware服务都是一项有价值、有意义的活动。

X-ware管理条例：内部工作条例

预约管理制度

  • （待定）只接受提交了预约的求助；
  • （待定）预约表的内容应包括：姓名、学院、预计前来时间、问题描述、联系方式（QQ/微信）；
  • 每次X-ware服务的预约时间为上周四17:00:00-本周四16:59:59，这一情况应在预约页面上说明；
  • 系统维护组与媒体运营部的管理团队负责预约表的处理；
  • （待定）每周四下午19:00前，媒体运营部将预约表发送给系统维护组；
  • （待定）系统维护组应与预约人取得联系，确认是否前来。

    服务引导

  • 每周安排一名同学负责引导和签到；
  • 在X-ware正式开始前，准备好签到表，预先写好预约人的信息，到达后预约人只需确认签名即可；
  • 原则上，负责签到的同学不应该参与到服务中。

    服务秩序

  • 工作位置间隔出来的位置提供给前来的同学；
  • 在工作过程中，尽可能不要将工作转交给其他同学（技术问题不在此列），不要冷落同学；
  • 在操作过程中，尽可能做出必要的解释；
  • 每个成员可以同时负责不超过两台的设备。

    注意事项

  • 尊重他人隐私，在操作过程中不进入无关的个人文件夹、文件备份前必须征得同意，操作根据附录中备份条例进行；
  • 谨慎操作，不得使用“移动”功能，使用“复制”+“删除”，在进行删除文件、格式化磁盘、删除分区等操作前至少向所有者确认三次；
  • 拆机前向所有者确认至少三次，说明可能带来的后果；
  • 不强行解决已知超过能力范围的问题；
  • 禁止向同学提供（线上和线下，尤其是线上）各类不符合版权规定的软件安装包/源代码；
  • 在对外宣传时禁止宣称提供不符合版权规定的软件安装服务。

    X-ware管理条例：对外服务流程

  1. 预约：前来求助的同学，应该首先通过微信公众号X-ware预约平台进行预约；
  2. 确认：在服务开始前，会有系统维护组成员向预约的同学确认是否前来；
  3. 签到：在X-ware登记表上确认信息无误后，签字进行确认；
  4. 负责引导的同学将前来的同学引导到对应的SNC成员处；
  5. SNC成员提供服务，如果求助同学需要提前离开，提供服务的同学应确认该同学的个人信息和联系方式，以保证可以取得联系。

    附录

    关于数字校园激活客户端的使用

    数字校园的激活客户端需要使用 .Net框架，在Windows 10上，这不是一个默认安装的组件，导致在使用客户端时必须要联网安装该框架。实际上，该框架的联网安装在校园网环境内较为困难，因此提出了下面的解决方法：
  • 需要用到的程序：Windows 10、DISM++、对应版本的Windows 10安装镜像
  • 首先，挂载Windows 10的安装镜像；
  • 启动DISM++，在程序和功能中找到Windows功能；
  • 勾选 .Net Framework 3.5，并在本地源中选择镜像目录下的/sources/sxs目录，点击应用；
  • 安装进度条显示在左上，完成后退出。

完成了上述处理之后，就可以使用激活客户端了。

关于备份的有关规定

X-ware服务过程中产生的零时备份文件，应该存储于SNC所有的存储介质中，在恢复完成之后当面删除备份文件；该介质严禁外借。
